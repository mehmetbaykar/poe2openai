Feature Guide

Model State Settings

  • No Mark (-): Keep original model settings without any modifications
  • Yellow (R): Compatibility handling primarily for Poe platform's text-to-image/video generation models
  • Crossed (X): Disable the model; it will not be shown in the models list

Model Mapping Feature

  • Set a new display name for models; the original name will be shown on the left with a strikethrough
  • Useful for mapping model names to other API formats
  • Mapped names will be used across all API endpoints

Global Enable Switch Impact

  • When enabled, affects the following API endpoints:
  • Model List Filtering & Mapping
  • - GET /v1/models
  • - GET /models
  • Chat Event Handling & Mapping
  • - POST /chat/completions
  • - POST /v1/chat/completions

Custom Models Feature

  • Add custom models to the models list
  • Simply enter the Bot name to add
  • Optionally provide model provider information
  • Custom models are marked with a green border

Search and Filtering

  • Use the search box to quickly find specific models
  • Filter by model type: All, Custom, Mapped, Replace Response, or Disabled
  • Toggle hide disabled models to improve interface clarity
  • Interface settings are automatically saved locally
